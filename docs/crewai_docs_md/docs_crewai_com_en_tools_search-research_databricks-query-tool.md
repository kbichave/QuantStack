# [​](https://docs.crewai.com/en/tools/search-research/databricks-query-tool\#databricksquerytool)  `DatabricksQueryTool`

## [​](https://docs.crewai.com/en/tools/search-research/databricks-query-tool\#description)  Description

Run SQL against Databricks workspace tables with either CLI profile or direct host/token authentication.

## [​](https://docs.crewai.com/en/tools/search-research/databricks-query-tool\#installation)  Installation

Copy

Ask AI

```
uv add crewai-tools[databricks-sdk]
```

## [​](https://docs.crewai.com/en/tools/search-research/databricks-query-tool\#environment-variables)  Environment Variables

- `DATABRICKS_CONFIG_PROFILE` or (`DATABRICKS_HOST` \+ `DATABRICKS_TOKEN`)

Create a personal access token and find host details in the Databricks workspace under User Settings → Developer.
Docs: [https://docs.databricks.com/en/dev-tools/auth/pat.html](https://docs.databricks.com/en/dev-tools/auth/pat.html)

## [​](https://docs.crewai.com/en/tools/search-research/databricks-query-tool\#example)  Example

Code

Copy

Ask AI

```
from crewai import Agent, Task, Crew
from crewai_tools import DatabricksQueryTool

tool = DatabricksQueryTool(
    default_catalog="main",
    default_schema="default",
)

agent = Agent(
    role="Data Analyst",
    goal="Query Databricks",
    tools=[tool],
    verbose=True,
)

task = Task(
    description="SELECT * FROM my_table LIMIT 10",
    expected_output="10 rows",
    agent=agent,
)

crew = Crew(
    agents=[agent],
    tasks=[task],
    verbose=True,
)
result = crew.kickoff()

print(result)
```

## [​](https://docs.crewai.com/en/tools/search-research/databricks-query-tool\#parameters)  Parameters

- `query` (required): SQL query to execute
- `catalog` (optional): Override default catalog
- `db_schema` (optional): Override default schema
- `warehouse_id` (optional): Override default SQL warehouse
- `row_limit` (optional): Maximum rows to return (default: 1000)

## [​](https://docs.crewai.com/en/tools/search-research/databricks-query-tool\#defaults-on-initialization)  Defaults on initialization

- `default_catalog`
- `default_schema`
- `default_warehouse_id`

### [​](https://docs.crewai.com/en/tools/search-research/databricks-query-tool\#error-handling-&-tips)  Error handling & tips

- Authentication errors: verify `DATABRICKS_HOST` begins with `https://` and token is valid.
- Permissions: ensure your SQL warehouse and schema are accessible by your token.
- Limits: long‑running queries should be avoided in agent loops; add filters/limits.
