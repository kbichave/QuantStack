# P10 Self-Interview: Meta-Learning & Self-Improvement

## Q1: How do you measure agent quality when outcomes are delayed (trade results take days/weeks)?
**A:** Two-tier measurement: (a) immediate quality — structural quality of the output (completeness, consistency, use of available data), scored via rubric; (b) outcome quality — direction accuracy, magnitude accuracy when trade resolves. Immediate quality is available same-cycle, outcome quality joins back when the position closes. Both contribute to the rolling score.

## Q2: How does the few-shot library avoid stale examples?
**A:** Examples are tagged with regime and date. Selection criteria: (a) regime match (current regime = example regime), (b) recency decay (newer examples preferred), (c) quality threshold (only top 10% by quality score). Max 3 examples per agent per context type. Examples older than 90 days are automatically retired.

## Q3: What prevents the meta-allocator from overfitting to recent performance?
**A:** Three safeguards: (a) minimum 63-day lookback window (quarterly data), (b) regularization in the linear regression (L2 penalty), (c) weight floor — no strategy gets less than 5% allocation if it's active. Monthly retraining with expanding window prevents chasing recent performance.

## Q4: How does research prioritization interact with the existing research queue?
**A:** It replaces FIFO ordering. Each research queue entry gets a priority score: expected_alpha_uplift (0-1) + portfolio_gap (0-1) + failure_frequency (normalized) + staleness (time decay). The weighted sum determines processing order. Weights are configurable. The queue consumer processes highest priority first.

## Q5: What's the relationship between P10 meta-allocator and P09 StrategySelectEnv?
**A:** They solve the same problem with different methods. P10's meta-allocator is a simple linear regression (interpretable, fast, low data requirement). P09's RL environment is a neural network (more flexible, higher data requirement, less interpretable). The plan: start with P10's linear model, introduce P09's RL model as a challenger via A/B testing (P03 framework).

## Q6: How do prompt variants get created?
**A:** Manual creation by the research pipeline. The system identifies underperforming agents (win rate < 50%), generates 2-3 prompt variants with specific changes (more context, different reasoning structure, additional constraints). Variants are NOT auto-generated by LLM — they're crafted based on failure analysis.

## Q7: What happens when the meta-allocator wants to allocate 0% to an active strategy?
**A:** It can't. The weight floor is 5% for any active strategy. If the meta-allocator consistently underweights a strategy (below 10% for 4 consecutive weeks), it triggers a strategy lifecycle review — the strategy may be demoted or retired through the existing P00 lifecycle pipeline. The meta-allocator doesn't have authority to retire strategies.

## Q8: How do you avoid circular feedback (agent quality score influences prompts, which influences quality score)?
**A:** Temporal separation. Quality scores use a 21-day rolling window. Prompt changes take effect immediately but are evaluated on NEW data only (post-change observations). A/B test comparison requires minimum 14 days of post-change data before promotion decision. This prevents the feedback loop from being self-reinforcing within a short window.
