# Prompt Optimization & Per-Trade Reward Learning

## Prerequisites

This stack requires **sufficient trade volume** before autonomous loops produce signal, not noise:

| Milestone | What's active |
|-----------|--------------|
| **Now (0-50 trades)** | ReflexionMemory, CreditAssigner (heuristic), HypothesisJudge, TextGrad (trade critique) |
| **200+ trades** | Consider OPRO with human-reviewed promotion (not autonomous) |
| **500+ trades** | Enable autonomous OPRO weekly loop with measurement |
| **1000+ trades** | Consider TrajectoryEvolution monthly |

Sharpe ratio on 5-10 trades has standard error ~1.0. Scoring prompt fitness on a week of trading at current volume is noise. **Do not enable autonomous prompt evolution until measurement infrastructure proves it works.**

## Architecture (Active)

```
Trade Close
    ├──▶ ReflexionMemory        Classify root cause, store episode
    ├──▶ CreditAssigner         Attribute loss to specific step (heuristic)
    ↓
AutonomousRunner
    └──▶ DebateFilter           Inject past lessons as bear evidence (+0.10/lesson, cap +0.30)

Nightly
    ├──▶ HypothesisJudge        Gate hypotheses before expensive backtests
    └──▶ TextGradOptimizer      Critique losing trades, store proposals

After each ML experiment
    └──▶ Judge.update_knowledge  Outer loop: backtests update judge KB
```

## Architecture (Dormant — code exists, not wired)

```
OPROLoop                    Weekly prompt evolution via meta-optimization
TrajectoryEvolution         Monthly crossover of research run segments
Langfuse tracing            Observability for optimization LLM calls
```

All optimization runs **offline** (after market close). The live
execution path (SignalEngine → DecisionRouter → RiskGate → Broker)
remains deterministic and LLM-free.

## Prompt Lifecycle

```
proposed ──▶ testing ──▶ champion ──▶ retired
    │            │
    │            └── Auto-rollback if Sharpe drops >1σ
    └── Generated by TextGrad or OPRO
```

- **proposed**: Stored critique/suggestion, not yet applied
- **testing**: Actively used in paper trading for 1 week minimum
- **champion**: Active prompt used by the system
- **retired**: Replaced by a better variant

## Integration Points

| Hook | Where | What it triggers |
|------|-------|------------------|

## Root Cause Taxonomy

| Root Cause | Trigger | Bear Penalty |
|------------|---------|-------------|
| `regime_shift` | Entry regime ≠ exit regime | +0.10 per lesson |
| `sizing_error` | Severe loss + high conviction | +0.10 per lesson |
| `entry_timing` | Signals present but trade moved against | +0.10 per lesson |
| `data_gap` | Missing signal data at entry | +0.10 per lesson |
| `strategy_mismatch` | Momentum in ranging / reversion in trending | +0.10 per lesson |
| `stop_loss_width` | Severe loss + low conviction | +0.10 per lesson |

Bear penalties from reflexion episodes are capped at +0.30 total.

## Safety Guarantees

1. **Risk gate is immutable** — optimization never touches `risk_gate.py` or `kill_switch.py`
2. **Execution path stays LLM-free** — all optimization is offline (post-market close)
3. **Paper mode for all experiments** — new prompts tested on paper for 1 week minimum
4. **Proposals, not auto-applies** — TextGrad critiques require OPRO validation or human review
5. **Audit trail** — every verdict, critique, and promotion logged in DuckDB
6. **Auto-rollback** — champion reverts if Sharpe drops >1σ below baseline
